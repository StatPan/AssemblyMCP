# AssemblyMCP Handover Guide: The Art of Data Connection

이 문서는 'PlayMCP' 대회 우승을 위해 AssemblyMCP가 지향해야 할 **"데이터 연결의 철학"**과 다음 단계를 기술합니다.

## 🌟 핵심 성과: "단순 조회에서 통찰(Insight)로"
기존의 서버가 API를 래핑하는 데 그쳤다면, 현재는 여러 API의 데이터를 엮어 새로운 가치를 창출하고 있습니다.

### 1. 연결의 예시: `get_representative_report`
- **Before**: 의원 인적사항 따로, 법안 따로, 경력 따로 조회 (에이전트가 3~4번 호출)
- **After**: 한 번의 호출로 **[의원 기본 정보 + 전문 분야(위원회 기간 계산) + 최근 표결 성향]**을 통합 리포트로 제공.
- **의미**: "3선 의원"이라는 정보보다 "기재위 67개월 활동한 경제 전문가"라는 연결된 정보가 사용자에게 더 큰 가치를 줍니다.

### 2. 기술적 정복: API 불일치 해결
- 국회 API의 `AGE` 파라미터가 서비스마다 다른 형식(`22` vs `100022`)을 요구하는 문제를 `_normalize_age_for_api` 로직으로 정규화하여 데이터 유실을 막았습니다.

## 🚀 다음 작업자를 위한 미션 (Next Connections)

다음 작업자는 아래와 같은 **"의미 있는 연결"**을 추가로 시도해 보길 권장합니다.

### 1. 의원 + 예산 (NABO Connection)
- **아이디어**: 특정 의원이 발의한 법안과 국회예산정책처(NABO)의 비용추계서 데이터를 연결.
- **가치**: "이 의원의 법안이 국가 재정에 어떤 영향을 미치는지" 분석 가능.

### 2. 위원회 + 뉴스 (Contextual Insight)
- **아이디어**: 위원회 소관 법안과 해당 기간의 '국회뉴스ON' 데이터를 연결.
- **가치**: "왜 이 법안이 이 위원회에서 쟁점이 되었는지" 언론의 시각을 포함한 입체적 답변 가능.

### 3. 법안 + 투표 + 정당 (Responsibility Tracking)
- **아이디어**: 특정 이슈(예: 간호법) 관련 모든 법안의 투표 결과를 정당별로 집계.
- **가치**: "이 이슈에 대해 각 정당은 어떤 역사적 태도를 취해왔는가?"에 대한 시계열 답변 가능.

## ⚠️ 기술적 팁 (Technical Tips)
- **SmartService가 두뇌입니다**: 새로운 복합 기능은 `smart.py`에 추가하고, 로우 레벨 API 호출은 `services.py`에서 담당하게 하세요.
- **병렬 처리를 잊지 마세요**: `asyncio.gather`를 사용하여 복합 데이터를 조회할 때 사용자 대기 시간을 최소화해야 합니다.
- **토큰 효율성**: Pydantic의 `model_dump(exclude_none=True)`를 적극 활용하여 에이전트에게 꼭 필요한 데이터만 전달하세요.

**"데이터는 흩어져 있을 때 숫자에 불과하지만, 연결될 때 정책이 됩니다."** 이 정신을 이어가 주시기 바랍니다.
